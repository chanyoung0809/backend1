서버(server)란? * 클라이언트의 요청을 받으면 서비스(웹 페이지), 데이터를 제공하는 컴퓨터 혹은 프로그램을 지칭함 - 서빙알바 : (음식을) 전달해줌, 전달해주는 사람 - 웹 브라우저(클라이언트(갑)) 프로그램을 이용해서 주소 -> naver.com, daum.net.... 각 페이지에서 해당 페이지 보여달라고 요청(웹툰, 블로그...) - 요청 request-> <-응답 response (서버(컴퓨터)(지식인, 블로그, 사진 ....)) - 웹 브라우저를 통해서 주소를 입력하면 원하는 페이지를 서버에서 여러분의 웹 브라우저를 통해 보여줌(rendering이라고 함) - 서버에 요청할 수 있는 작업 방법 4가지 1. 읽기(웹페이지를 읽을 때) read 2. 쓰기(글작성 / 로그인 / 회원가입 등.. 없는 걸 만들어냄) create 3. 수정(게시글 수정, 배송정보 수정, 회원정보 수정...) update, put 4. 삭제(게시글 삭제, 회원탈퇴...) delete - form 태그는 계속 사용됨
폼 태그 관련 태그들 세부사항 잘 기억하기.
- 아이디, 비밀번호, 게시글 제목, 내용... 등 중요도 높고 주소창에 노출되지 않아야 하는 것(노출될 경우 악의적으로 이용될 수 있음) -> post - 검색창의 데이터, 상세페이지 등 수많은 페이지 중 원하는 것을 보고자 할 때 요청 -> get
